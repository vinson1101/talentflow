# 招聘决策摘要卡片 V1

## 定位
本模板用于生成人类可读的招聘决策摘要（展示层产物）。

> **注意：**
> - **系统真相**：`final_output.json`、飞书 Runs / Candidates 字段
> - **展示层产物**：本摘要卡片
> - 本模板不替代评分规则，不替代输出 contract

---

## 标准区块（5个）

### 区块1：标题
```
【招聘决策报告】{岗位名称} · {城市} · {薪资范围}
```

### 区块2：批次概览
```
本批共处理 {总人数} 份简历，决策结果：
```

### 区块3：分档表

| 档位 | 人数 | 候选人 |
|---|---:|---|
| 🟢 A / strong_yes（强烈推荐，今天联系） | {A人数}人 | {A候选人列表} |
| 🔵 B / yes（值得联系，本周联系） | {B人数}人 | {B候选人列表} |
| 🟡 C / maybe（备选观察，可选联系） | {C人数}人 | {C候选人列表} |
| ⚫ no（不推进） | {No人数}人 | {No候选人列表} |

**说明：**
- 候选人可写为 `姓名（分数）`
- 人数不固定
- 不要求每批都必须有 A 类或 strong_yes
- no 档可省略（若无不推进候选人）

### 区块4：今日优先联系 / 本周建议联系

**今日优先联系：**
- **{候选人1}**（{分数}分 / {档位}）：{一句话核心判断}。风险：{一句话风险}
- **{候选人2}**（{分数}分 / {档位}）：{一句话核心判断}。风险：{一句话风险}

**本周建议联系：**
{候选人列表}（重点核实：{验证点}）

**说明：**
- 不写死 "Top N"
- 具体人数由结果决定
- 若无人需要今日优先联系，该节可为空或写"无"
- 本周建议联系以 B 档为主

### 区块5：主要风险提示

- {风险1}
- {风险2}
- {风险3}

**说明：**
- 风险是批次级总结，不是单人重复描述
- 优先提炼最影响推进效率的风险

## 示例

```
【招聘决策报告】高级产品经理 · 杭州 · 11-15K

本批共处理 12 份简历，决策结果：

| 档位 | 人数 | 候选人 |
|---|---:|---|
| 🟢 A / strong_yes（强烈推荐，今天联系） | 2人 | 邢威峰（87.4分）、李蓓（81.2分） |
| 🔵 B / yes（值得联系，本周联系） | 2人 | 孙铜（64.7分）、吴嘉楠（63.0分） |
| 🟡 C / maybe（备选观察，可选联系） | 6人 | 姜月、唐晓斌、王宏宇、许罗栋、周俊仕，张金莎 |
| ⚫ no（不推进） | 2人 | 滕锋帅（53.5分）、杨淑婷（35.5分） |

**今日优先联系：**
- **邢威峰**（87.4分 / A）：主导互联网医疗产品与医疗器械注册，5年B端产品经验。风险：最近一份工作仅2个月离职。
- **李蓓**（81.2分 / A）：城市大脑数字驾驶舱项目，跨多局委办协调能力强。风险：简历截止较早，需核实现状。

**本周建议联系：**
孙铜、吴嘉楠（重点核实：稳定性与产品 ownership 深度）

**主要风险提示：**
- 部分候选人存在履历时间断档，需核实当前在职状态
- B档候选人产品 ownership 深度需首轮沟通确认
- C档多为转岗/经验不足，适合作为观察池

**run_id：** run_20260324_194538
**生成时间：** 2026-03-24 19:47
```
